<h1>Old maid ババ抜き（Java）</h1>
- 開発期間：約13時間（2025年11月～12月）<br>
- 言語：Java 21<br>
- 実行環境：コマンドプロンプト<br>

<br>
Javaのオブジェクト指向などの基礎知識を確認に作成した<br>
1対1ババ抜きコンソールアプリケーション（CUI）です。<br>
<br>
<b>-特徴・こだわりポイント-</b><br>
・1対1に絞り完璧に作ることを優先しました。<br>
<br>
・友達にプレイしもらい、手札を捨てるときの手札番号が見辛いと指摘があり、<br>
手札の文字列の書式を設定しました。<br>
・職業訓練校の先生にも見て頂き、hashcode()のオーバライドやコードの書き方を<br>
修正しました。<br>
<br>
・継承を使用→人間のプレイヤークラスHumanとコンピュータープレイヤークラス<br>
ComputerにPlayerインターフェースを実装しました。インターフェースには<br>
HumanとComputerに共通するメソッドを記載しました。<br>
・多態性を使用→上記のHumanクラスとComuterクラスをPlayerの型で使用しました。<br>
・カプセル化の実現→すべてのクラスのフィールドをprivateで修飾しメソッドは<br>
publicで修飾しカプセル化をしました。<br>
<br>
・将来的に手動でカードを作成する場合等に備え、Cardのセッターには、<br>
値の妥当性を検査する機能を実装しました。<br>
・ユーザーがコンピューターの手札を引くときには、ユーザーの誤入力や将来の改修<br>
によるバグを考慮し、「事前チェック（if文）」＋「例外処理による安全網」<br>
の二段構えで防御しています。<br>
<br>
<b>-学んだこと-</b><br>
・動くようになったら、友達にプレイしてもらいましたが見辛いと指摘あり、<br>
ユーザーの声で改善する大切さを学びました。<br>
・カプセル化を最初Cardクラスしかしておらず、<br>
オブジェクト指向の基本カプセル化の徹底の大切さを学びました。<br>
・equals()をオーバーライドしたら、hashCode() もオーバーライドする<br>
大切さを学びました。<br>
<br>
<b>-実行方法-</b><br>
Windowsでスート記号（♠♥♦♣）を正しく表示するには<br>
コマンドプロンプトで以下の通りに実行してください。<br>
<br>

chcp 65001<br>
